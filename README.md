# 東美レジ v2 — オフライン対応 POSレジ PWA

iPadのSafariで開いて「ホーム画面に追加」すると、他のアプリと同じようにアイコンタップで全画面起動するレジアプリです。レジ機能は完全オフラインで動作します。

## v2 での変更点（前バージョンとの違い）

| 項目 | 内容 |
|---|---|
| ★ 商品ごとの割引 | カゴ内の商品ごとに **金額値引き（円）** と **％割引** の両方が使える |
| ★ AI価格読み取り | 価格変更表を撮影 → AIが商品名と新価格を読み取り → 確認して一括反映 |
| − バーコードスキャン | 廃止（シンプル化） |
| − 在庫管理 | 廃止（シンプル化） |

## 機能一覧

- **レジ**：商品タップでカゴ追加、数量変更、商品ごとの値引き/割引、現金会計、クイック金額ボタン、お釣り自動計算
- **商品管理**：登録・編集・削除、カテゴリ、価格（税込）
- **AI価格読み取り**：価格変更表の写真から商品名＋新価格を抽出 → プレビューで対象商品と適用行を確認 → 一括更新
- **画面レシート**：取引No・日時・担当・明細・割引・内税を画面表示（紙印刷なし）
- **売上履歴・返品**：取引一覧、タップで明細、返品・返金処理
- **集計**：本日/今月/累計の売上、カテゴリ別・スタッフ別内訳（期間切替）
- **スタッフ管理**：複数登録、担当切替（ヘッダーからも切替可）、スタッフ別売上記録
- **CSV出力**：商品マスタ・売上履歴（UTF-8 BOM付き、Excelで文字化けなし）

## iPadへの導入手順（GitHub Pages 推奨）

PWAの完全オフライン動作には **HTTPS環境が必須**（Safariの仕様）です。

1. GitHubの新規リポジトリ（例: `tobi-pos`）に全6ファイルをアップロード
2. リポジトリ設定 → Pages → ブランチ `main`・フォルダ `/` で公開
3. 数分後 `https://ユーザー名.github.io/tobi-pos/` で公開される
4. iPadのSafariで開く → 共有ボタン → **「ホーム画面に追加」**
5. ホーム画面の「東美レジ」アイコンから起動 → 全画面アプリとして動作
6. 初回起動時に全アセットがキャッシュされ、以降は完全オフラインで動作

※ 前バージョンと同じリポジトリに上書きアップロードしてもOKです（データベース名が `tobi-pos-v2` に変わっているため、旧版のデータとは別管理になります。旧版の売上はアップ前にCSV出力しておいてください）。

## AI価格読み取りの使い方

1. **設定タブ**で Anthropic APIキー（`sk-ant-...`）を保存
   - キーは https://console.anthropic.com で取得（従量課金。読み取り1回あたり数円程度）
   - キーはiPad内（IndexedDB）にのみ保存されます
2. **商品タブ** → 「写真を読み取る」→ 価格変更表を撮影
3. 読み取り結果のプレビューで、対象商品（自動マッチ・手動選択可）と適用行のチェックを確認
4. 「価格を更新」で一括反映

⚠️ **この機能の実行時のみインターネット接続が必要**です。レジ・会計・履歴・集計はすべてオフラインで動きます。

### 読み取り精度を上げるコツ

- 表全体が枠に収まり、ピントが合った写真を使う
- 商品マスタの商品名と価格変更表の表記をなるべく揃えておく（自動マッチの精度が上がります）

## 割引の使い方

カゴ内の商品の「値引き / 割引」ボタン → 金額値引き（円）か％割引を選択 → クイックボタンまたは手入力 → 適用。レシート・履歴・CSVにも割引内容が記録されます。

## データの保存場所と注意

すべて **iPadのブラウザ内（IndexedDB）** に保存されます。

- Safariの「サイトデータを消去」で全データが消えます
- **定期的にCSV出力でバックアップ**してください
- 1台運用前提（複数端末間の同期はありません）

## 既知の制約

- 紙レシート印刷なし（画面表示のみ）
- 決済は現金のみ
- 税率は10%内税のみ（軽減税率8%は未実装）
